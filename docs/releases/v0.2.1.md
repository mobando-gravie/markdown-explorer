# Markdown Explorer v0.2.1

A patch release fixing the syntax highlighter integration.

## Fixed

- **Code blocks weren't highlighting** in v0.2.0. The custom code-block view rendered the raw source through `Text(AttributedString)` directly, bypassing MarkdownUI's text-style pipeline so the per-run colours from highlight.js never made it to screen.
- Wired the canonical `markdownCodeSyntaxHighlighter(.highlightJS(colorScheme:))` modifier and switched the custom block view to render `configuration.label`, which goes through the highlighter and respects the syntax colours.

## Install

1. **Download** `MarkdownExplorer-v0.2.1.dmg` below.
2. **Open** the DMG and drag the app to **Applications**.
3. **First launch** — macOS will warn that the app isn't notarized (this build is ad-hoc signed). Right-click the app → **Open** → **Open** in the dialog. macOS remembers the choice. Or use **System Settings → Privacy & Security → Open Anyway** after the first refused launch.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
