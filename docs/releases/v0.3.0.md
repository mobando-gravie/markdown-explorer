# Markdown Explorer v0.3.0

A renderer-replacement release. The big one: select rendered text and paste with formatting preserved into emails, Jira, Confluence, Notion, TextEdit, Word, and anywhere else that accepts HTML.

## New

- **Rich-text copy** — pick any portion of the rendered preview, hit ⌘C, then paste into Gmail, Apple Mail, Jira, Confluence, TextEdit RTF, Word, Pages, or Notion. Bold, italics, links, headings, lists, tables, syntax-highlighted code, and inline code all keep their styling. The pasteboard now carries both `text/plain` and `text/html` payloads.
- **Light/Dark toggle** — a sun/moon button in the toolbar that flips the entire app, persisted across launches. First-launch seed comes from your system appearance; after that, your choice sticks.

## Changed

- **Markdown rendering** moved from `swift-markdown-ui` (SwiftUI) to a `WKWebView`-based renderer. Markdown parsed by `markdown-it`, syntax-highlighted by `highlight.js`, and Mermaid diagrams rendered by `mermaid.js` — all inside a single HTML page styled with `github-markdown-css`.
- **Trade-off**: the rendered look is now GitHub-style markdown rather than the previous SwiftUI theme. Functionally similar, visually a touch different.

## Removed

- `swift-markdown-ui` dependency — no longer needed; rendering happens in the WKWebView.
- The Swift-side `HighlightEngine` JSC adapter, `HighlightedCodeBlock` view, and the standalone `MermaidWebView` — folded into the unified page renderer.

## Install

1. **Download** `MarkdownExplorer-v0.3.0.dmg` below.
2. **Open** the DMG, drag **Markdown Explorer** to **Applications**.
3. **First launch** — the build is still ad-hoc signed (no paid Apple Developer ID), so macOS will refuse to open it with the "cannot check for malicious software" warning. Right-click → **Open** → **Open** (or **System Settings → Privacy & Security → Open Anyway** after the first refused launch). macOS remembers your choice.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
