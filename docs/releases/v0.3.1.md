# Markdown Explorer v0.3.1

A polish release that fixes v0.3.0's blank renderer and ships a proper, native rendering pipeline.

## Fixed

- **Blank document area in v0.3.0** — WKWebView wrapped via `NSViewRepresentable` wouldn't render content. Replaced with a native `NSTextView` pipeline.

## New / improved

- **Real GFM rendering** via [Apple's swift-markdown](https://github.com/swiftlang/swift-markdown) parser walking the AST and emitting a fully styled `NSAttributedString`. Headings, paragraphs, bold / italic / strikethrough, links, inline code, fenced code blocks, blockquotes, ordered + unordered + task lists, **GFM tables**, thematic breaks, and inline images all render correctly.
- **Syntax-highlighted code blocks** with proper token colouring (~38 languages via `highlight.js` in `JavaScriptCore`).
- **Mermaid diagrams** render inline via an offscreen `WKWebView` snapshot → `NSImage` → `NSTextAttachment`. A "Diagram rendering…" placeholder appears briefly while the snapshot resolves.
- **Rich-text copy preserved** — ⌘C on any selection puts plain text, RTF, and HTML on the pasteboard; pasting into Gmail / Jira / Confluence / Apple Mail / TextEdit RTF / Pages / Word picks up formatted output.
- **GitHub-style polish** — sized headings with bottom borders on h1 / h2, padded code-block backgrounds, indented blockquotes with muted text, table cells with borders + tinted header row.
- **Light/Dark toggle** continues to work via the toolbar sun/moon button.
- **⌘F** opens the NSTextView native find bar inside any rendered document.

## Install

1. **Download** `MarkdownExplorer-v0.3.1.dmg` below.
2. **Open** the DMG, drag the app to **Applications**.
3. **First launch**: build is ad-hoc signed; macOS will warn. Right-click the app → **Open** → **Open** (or **System Settings → Privacy & Security → Open Anyway** after the first refused launch).

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
