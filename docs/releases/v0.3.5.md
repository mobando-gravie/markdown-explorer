# Markdown Explorer v0.3.5

## Fixed

- **Code blocks with no language tag now render unstyled.** Before, an unlabelled fence (` ``` `) ran through `hljs.highlightAuto(...)`, which would guess a language and start colouring tokens — for ASCII-art diagrams that meant the pipe and arrow characters picked up the *keyword* colour, leaving a visible pink/magenta box outline. Now an empty or unknown language returns plain monospaced text immediately, matching GitHub's behaviour. Code fences that *do* declare a language (`swift`, `javascript`, `python`, etc.) still go through highlight.js as before.
- **Blockquotes now render with a tinted background and a left bar**, matching the GitHub markdown style. Built with an `NSTextBlock` so the rectangle spans the full column width instead of only the glyph runs.

## Install

1. **Download** `MarkdownExplorer-v0.3.5.dmg` below.
2. **Open** the DMG, drag the app to **Applications**.
3. **First launch**: ad-hoc signed, so macOS will warn. Right-click → **Open** → **Open**, or **System Settings → Privacy & Security → Open Anyway** after the first refused launch.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
