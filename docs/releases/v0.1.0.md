# Markdown Explorer v0.1.0 — first release

A native macOS SwiftUI app for browsing and reading folders of Markdown files. File tree on the left, rendered preview on the right, plus a smart "In progress" view that surfaces the projects you've recently touched.

## What's in this build

- **Two sidebar modes** — *Explore* (lazy recursive file tree) and *In progress* (historical project list with hot/cold polling, sorted by most-recent edit).
- **GFM rendering** — headings, lists, task lists, tables, blockquotes, syntax-highlighted code, links, images.
- **Mermaid diagrams** — ` ```mermaid ` fenced blocks render live; bundled engine, no network required at runtime.
- **Auto-detects project roots** — walks ancestors looking for `.git`, `package.json`, `Cargo.toml`, `pyproject.toml`, `go.mod`, `Package.swift`, `node_modules`, `.idea`, `.vscode`, `Pods`, `vendor`, and ~20 other markers to label rows as `project › parent`.
- **Sandboxed**, with security-scoped folder bookmarks so the last folder reopens automatically.

## Install

1. **Download** `MarkdownExplorer-v0.1.0.dmg` from the assets below.
2. **Open** the DMG (double-click) and drag **Markdown Explorer** to the **Applications** folder shortcut inside the DMG window.
3. **First launch** — macOS will refuse to open the app with a "Apple cannot check it for malicious software" warning. This is expected: this release is ad-hoc signed (no paid Apple Developer ID).

   To bypass once:
   - In Finder, **right-click** (or Control-click) the app → **Open** → click **Open** in the dialog.
   - macOS remembers the choice; subsequent launches are normal.
   - Alternative: **System Settings → Privacy & Security → "Open Anyway"** after the first refused launch.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
