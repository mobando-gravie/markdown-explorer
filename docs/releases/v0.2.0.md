# Markdown Explorer v0.2.0

A roundup of fixes and quality-of-life features on top of v0.1.0.

## New

- **Syntax-highlighted code blocks** — fenced code blocks in ~38 languages now render with token colouring (Swift, JavaScript, Python, Go, Rust, JSON, HTML, CSS, shell, SQL, and more). Powered by highlight.js bundled in-app and run via `JavaScriptCore`. Light and dark themes follow your system appearance.
- **Copy button on code blocks** — hover over any code block to reveal a Copy button. Raw source goes to the clipboard.
- **Click-to-navigate markdown links** — internal links to other `.md` files (e.g. `[other](./other.md)`) now load that file in the preview. External links continue to open in your browser.
- **Quick Open (⌘⇧O)** — fuzzy-search any `.md` file in the workspace. Arrow keys to navigate, Enter to open, Esc to dismiss.
- **Reveal in Finder** — right-click any file row in the sidebar to jump to it in Finder.
- **Word count + reading time** — small subtitle on every rendered document: e.g. `342 words · 2 min read`.
- **Recent folders** — `File → Open Recent` keeps the last 5 workspaces (security-scoped bookmarks restored across launches).

## Fixed

- **Dock icon was blank** at full size — the asset catalog was emitting an `AppIcon.icns` capped at 256×256 because slot filenames were aliased. Catalog rebuilt with one PNG per slot; `Assets.car` now carries all sizes through 1024×1024.

## Install

1. **Download** `MarkdownExplorer-v0.2.0.dmg` below.
2. **Open** the DMG, drag **Markdown Explorer** to the **Applications** folder shortcut.
3. **First launch** — macOS will show a "cannot check for malicious software" warning (this build is still ad-hoc signed, no paid Apple Developer ID yet). One-time bypass:
   - In Finder, **right-click** the app → **Open** → click **Open** in the dialog.
   - macOS remembers the choice.
   - Or after the first refused launch: **System Settings → Privacy & Security → Open Anyway**.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
