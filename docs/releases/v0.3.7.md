# Markdown Explorer v0.3.7

## Fixed

- **Trailing white band below the In progress list after the app sits idle.** The list lived inside a `LazyVStack`, which caches estimated row heights. When the background tick (every 60s while In progress is active) rebuilt `wipFolders` and a folder's file list shrank, the cached height stayed stale and the `ScrollView` ended up taller than the actual content — leaving a blank strip below the last folder until you hit Refresh. Replaced with an eager `VStack`; rows are always measured directly, so tick-driven updates settle the same way a full refresh does. The list is small by design (each folder is capped at 10 file rows), so there's no real performance cost.

## New

- **⌘F now opens the in-document find bar.** The text view side was already configured for find (`usesFindBar = true`), but the matching menu/shortcut wasn't wired, so ⌘F was a no-op. Added Find / Find Next / Find Previous (⌘F / ⌘G / ⌘⇧G) under the Edit menu. Invoking ⌘F focuses the rendered document and slides in the native AppKit find bar — incremental highlights, ↩ for next match, full regex/case toggles for free.

## Install

1. **Download** `MarkdownExplorer-v0.3.7.dmg` below.
2. **Open** the DMG, drag the app to **Applications**.
3. **First launch**: ad-hoc signed, so macOS will warn. Right-click → **Open** → **Open**, or **System Settings → Privacy & Security → Open Anyway** after the first refused launch.

## Requirements

- macOS 14 Sonoma or later
- Apple Silicon (arm64)
